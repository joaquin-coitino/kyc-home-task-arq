1. Executive Summary

ARQ's KYC pipeline processed 52,075 user verification attempts between Jul 25, 2023 and Sep 22, 2023, achieving an overall pass rate of 86.1%. While the majority of users pass without friction, the analysis surfaces three material inefficiencies and several data quality issues that together represent a meaningful opportunity to improve conversion and reduce operational risk.

Finding 1 — Pipeline blockages account for the largest rejection category 2,977 (44.0%) of all rejections occur because the Extraction check never ran, stalling all downstream document checks. These are not genuine verification failures — they are infrastructure or UX problems and represent the highest-impact improvement opportunity.
Finding 2 — A 10-point pass-rate gap exists between Argentina (92.1%) and Mexico (81.5%) Mexican users are disproportionately rejected. The gap is partly explained by MEX National ID having an 89.3% pass rate vs. Electoral ID at 95.8% — suggesting document-guidance improvements could close this gap significantly.
Finding 3 — Rejection rate nearly doubled during Aug 21 – Sep 3 The weekly rejection rate spiked from a baseline of ~10–11% to 16–19% during this two-week window, then partially recovered. The root cause is unknown from the data alone and warrants investigation with the vendor.

About the data

Two datasets were provided: KYC_Summary (one row per attempt with date and outcome) and KYC_Details (one row per attempt with the result of each individual check). Both contain 52,075 unique user references with no duplicates.

Date range: Jul 25, 2023 – Sep 22, 2023 (59 days)

Countries: Mexico (MEX) — 29,706 attempts  |  Argentina (ARG) — 22,369 attempts

Document types: ID Card (44,638), Passport (2,660), Driving License (972), Visa (24)

KYC checks performed: Usability (root) → Extraction → Image Checks → Data Checks / Watchlist Screening; Liveness and Similarity run in parallel off Usability. See Section 3 for the full dependency graph.

3. Failure Funnel Analysis

The Jumio KYC pipeline is not a simple linear sequence — it is a dependency graph. Usability is the root check. Liveness and Similarity run in parallel off Usability and are independent of the document chain. Image Checks and Data Checks depend on both Usability and Extraction completing successfully.

Root cause of rejections

Key observations

Pipeline blockages — 2,977 rejections (44.0% of all rejections) These users were not rejected because a check explicitly failed — the pipeline itself stalled.
  • Extraction blocked (2,022): Usability passed but Extraction never ran, halting Image Checks, Data Checks, and Watchlist Screening.
  • Usability WARNING cascaded (530): A usability warning (e.g. blurred image) was treated as a hard blocker for Extraction.
  • Usability not executed (425): The root check itself did not run.
Similarity failures — 1,000 rejections (14.8%) Selfie does not match the document photo. Similarity depends on Usability only for face detectability, so it runs independently of the document chain.
Liveness failures — 908 rejections (13.4%) The system cannot confirm the user is live. Liveness is independent of the document chain — it runs on the selfie as soon as Usability passes. The most common cause is UNDETERMINED, suggesting connectivity or UX issues rather than spoofing.
Image check failures — 945 rejections (14.0%) Documents flagged as manipulated (537 cases) or digital copies (225 cases).
Usability hard rejections — 933 rejections (13.8%) The document image could not be processed at all (wrong document type, not uploaded, etc.). A usability rejection blocks the entire document chain.

4. Failure Reasons Deep Dive

Drilling into the details columns reveals the specific error codes behind each check failure, enabling targeted interventions.

Usability Failures

UNSUPPORTED_DOCUMENT_TYPE (509) and MISSING_MANDATORY_DATAPOINTS (407) are the top causes. Both can be reduced with better in-app guidance before document capture.

Image Check Failures

MANIPULATED_DOCUMENT (537) is the dominant failure reason and represents genuine fraud signals — these cases should be escalated for manual review. DIGITAL_COPY (225) may indicate users photographing a screen instead of a physical document.

Liveness Failures

liveness_UNDETERMINED (965) accounts for the vast majority. UNDETERMINED means the system could not reach a confident verdict — this is almost always a UX / connectivity issue, not spoofing. Retry flows would recover most of these users.

Similarity Failures

NO_MATCH (1,104) means the selfie and document photo could not be matched. NOT_POSSIBLE (408) means the check could not be performed — these users currently receive a WARNING outcome and still pass, which may need review from a compliance perspective.

5. Document Type Performance

Findings

Passports achieve the highest pass rate (96.4%) followed by Visas (95.8%). ID Cards (92.6%) and Driving Licenses (89.5%) perform below the overall average.

However, document volumes tell a different story: ID Cards account for 44,638 attempts (86%) of all attempts, making it the critical document type to optimise.

Sub-type variation

Within ID Cards, there is significant variation by sub-type — particularly in Mexico where Electoral IDs (95.8%) outperform National IDs (89.3%) by 6.5 percentage points. Guiding MEX users toward Electoral IDs could meaningfully improve their overall pass rate.

6. Country Analysis

The dataset covers two markets: Mexico (29,706 attempts, 57%) and Argentina (22,369 attempts, 43%). There is a 10.6 percentage-point gap in pass rates between them.

Why is Mexico's pass rate lower?

Document mix: Mexico has a higher share of National IDs (89.3% pass rate) relative to Electoral IDs (95.8%). If MEX users adopted Electoral IDs at the same rate as other sub-types, the estimated pass rate would improve by ~2–3pp.

RESIDENT_PERMIT_ID in Mexico has a 96.0% pass rate — one of the highest — suggesting the document quality standards are achievable.

Argentina's Driving Licenses (87.4%) are the weakest performer in that market, representing an opportunity for targeted guidance.

The remaining gap likely reflects differences in user demographics, device quality, internet connectivity, and potentially different fraud patterns.

7. Time Trends

Monitoring the rejection rate over time reveals a significant anomaly in late August / early September that warrants investigation.
Rejection spike: Aug 21 – Sep 3 The rejection rate jumped from a baseline of ~10.7% to 16.1% (Aug 21–27) and 18.7% (Aug 28–Sep 3) — a 1.8x increase. The rate partially recovered afterwards but remained above baseline (13–14%) through the end of the observation period.

9. Recommendations

Recommendations are ordered by estimated impact. Items 1–3 are high-priority and can be addressed within a single sprint; items 4–6 require cross-functional coordination.
1
Diagnose and fix pipeline blockages (Extraction not executing)

2,977 rejections (44% of all rejections) occurred because the Extraction check never ran — not because of a genuine verification failure. The three sub-causes are: Extraction blocked despite Usability passing (2,022), Usability WARNING cascading downstream (530), and Usability not executing at all (425). Investigate vendor API timeouts, upload failures, and whether Usability WARNINGs should truly block Extraction. Recovering even 50% of these cases would add ~1,488 passed users.

2
Guide Mexican users toward Electoral IDs

Electoral IDs pass at 95.8% vs. National IDs at 89.3% in Mexico. Add in-app guidance recommending Electoral ID as the preferred document. This is a low-effort UX change with a measurable impact on MEX pass rates.

3
Introduce a guided retry flow for liveness failures

Of the 1,032 liveness rejections, 965 are UNDETERMINED — meaning the system could not reach a conclusion, not that the user failed. These users should be prompted to retry in better lighting conditions rather than being hard-rejected.

4
Investigate the Aug 21 – Sep 3 rejection spike

The rejection rate nearly doubled during this period (from 10.7% to 18.7%). Cross-reference with vendor SLA reports, marketing campaigns, and app release history to identify the root cause and prevent recurrence.

5
Review watchlist warning handling

69 users triggered a watchlist warning but all 69 were marked as PASSED. Confirm with compliance that this is intentional and that a manual review process exists for these cases.

6
Enforce data quality at the pipeline level

Add schema validation to reject non-standard label values (OK, APPROVED, PASSES) and flag misrouted fields (liveness values in usability columns) in real time. This ensures analytics remain reliable as the pipeline evolves.
